## 1. 시스템 개요

AndanteFit ERP는 의료 관광 코디네이터가 해외 환자의 케이스를 처음부터 끝까지 관리하는 시스템입니다.

### 핵심 원칙

| 원칙 | 설명 |
|------|------|
| **케이스가 중심** | 모든 데이터 입력과 수정은 **케이스 상세** 화면에서 합니다 |
| **대시보드는 읽기 전용** | 공급관리·시술일정·결제청구·추적관리는 현황 조회만 가능 |
| **클릭으로 케이스 이동** | 대시보드에서 항목 클릭 시 해당 케이스 상세로 바로 이동 |

### 케이스 상태 흐름

```
리드 → New → Consulting → Acceptance Confirmed → Under Hospital Review
     → Treatment Scheduled → Treatment Done → Follow-Up → Closed
```

## 2. 로그인 및 권한

Google 계정으로 접속하면 자동 인증됩니다. 권한은 아래 세 가지입니다.

| 역할 | 주요 권한 |
|------|-----------|
| **MSO Admin** | 모든 기능 접근, 케이스 삭제, 팀원 관리 |
| **MSO Coordinator** | 케이스 생성·수정, 병원 검토 기록, 납부 처리 |
| **Hospital Staff** | 담당 케이스 조회, 병원 검토 결과 입력 |

## 4. 리드관리

### 목적
상담 문의가 들어온 잠재 고객(리드)을 관리합니다.

### 주요 작업

**리드 추가**
1. 우측 상단 **[+ 리드 추가]** 클릭
2. 이름, 연락처, 관심 시술 입력
3. **[저장]** 클릭

**리드 → 케이스 전환**
1. 리드 목록에서 해당 리드 클릭
2. **[케이스 전환]** 버튼 클릭
3. 병원, 코디네이터 배정 후 확인

> 케이스로 전환된 리드는 케이스 메뉴에서 계속 관리합니다.

## 5. 케이스 관리 (정보 입력의 중심)

케이스는 시스템의 핵심입니다. **모든 정보 입력과 수정이 여기서 이루어집니다.**

### 5-1. 케이스 목록

- 기본 화면: 모든 활성 케이스 표시
- 검색: 환자명, 케이스 ID, 코디네이터명
- 필터: 상태별 (리드/신규/상담 중/승인 확정/검토 중/시술 예정/시술 완료/추적 관리/종료)
- 각 행 클릭 → 케이스 상세로 이동

**케이스 ID 형식**: `MSO-YYYY-001` (예: `MSO-2026-042`)  
**화면 표시 형식**: `환자이름-번호` (예: `야마다타로-042`)

### 5-2. 케이스 상세 화면

케이스 상세는 **단일 스크롤 페이지**입니다. 탭 없이 모든 정보가 순서대로 표시됩니다.

#### 화면 구성 순서

```
① 헤더 (환자이름-번호, 병원, 코디네이터, 상태)
② 단계 표시바 (현재 단계 강조)
③ 케이스 기본정보 카드 + 환자 정보 카드
④ 병원 검토
⑤ 주문/배송
⑥ 결제/청구
⑦ 추적관리
⑧ 아카이브 (문서 + 활동 로그)
```

#### ③ 케이스 기본정보 인라인 수정

1. **[수정]** 버튼 클릭 → 입력 폼으로 전환
2. 수정 가능 항목:
   - 적응증 (target indication)
   - 우선순위 (Normal / High / Urgent)
   - 시술 예정일
   - 메모
3. **[저장]** → 즉시 반영, **[취소]** → 원래 값으로 복원

#### ④ 병원 검토 섹션

**검토 결과 빠른 입력** (Draft 또는 Under Hospital Review 상태일 때):
- **[✅ 병원 승인]** → 검토 결과 "Suitable" 즉시 기록
- **[❌ 진행 불가]** → 검토 결과 "Not Suitable" 즉시 기록
- **[📝 메모 포함 기록]** → 상세 메모와 함께 기록 (모달 팝업)

#### ⑤ 주문/배송 섹션

**공급 주문 등록**
1. **[+ 공급 주문]** 클릭
2. 입력 항목:
   - 공급업체 선택
   - 요청 품목
   - 수량
   - **병원 배송 예정일** (필수)
   - 메모
3. **[등록]** 클릭 → 자동으로 "배송 중" 상태로 등록됨

**출고 확정** (오프라인 배송 확인 후)
- 주문 행의 **[출고확정]** 버튼 클릭

> 배송중 확인 단계는 없습니다. 등록 즉시 배송 중으로 처리됩니다.

#### ⑥ 결제/청구 섹션

**청구 생성**
1. **[+ 청구 생성]** 클릭
2. 케이스 선택: 이름 또는 케이스 ID 입력 → 자동완성 목록에서 선택
3. 청구 금액 입력
4. **[생성]** 클릭

> 견적서·청구서 발행은 오프라인으로 처리합니다. 시스템에서는 납부 여부만 관리합니다.

**납부 처리**
1. 청구 항목의 **[납부 처리]** 클릭
2. 납부 유형 선택:
   - **일시납부**: 전액 납부 (금액 자동 입력)
   - **부분납부**: 납부 금액 직접 입력
   - **환불**: 환불 처리 (상태 → Refunded)
3. **[저장]** 클릭

#### ⑦ 추적관리 섹션

추적관리는 시술 후 D7, D14, D30, D90, D180 단계별로 환자 상태를 확인합니다.

**완료 처리**
1. 해당 추적관찰 항목의 **[완료 처리]** 클릭
2. 환자 반응 기록 (양호/재진필요/무응답/에스컬레이션)
3. 메모 입력 후 **[저장]**

### 5-3. 케이스 상태 변경 (단계 전환)

상세 화면 상단 액션 버튼으로 다음 단계로 전환합니다.

| 현재 상태 | 버튼 | 다음 상태 |
|-----------|------|-----------|
| New | [상담 시작] | Consulting |
| Consulting | [승인 확정] | Acceptance Confirmed |
| Acceptance Confirmed | [병원 검토 요청] | Under Hospital Review |
| Under Hospital Review | (검토 기록 후 자동) | Treatment Scheduled / Closed |
| Treatment Scheduled | [시술 완료] | Treatment Done |
| Treatment Done | [추적 시작] | Follow-Up |
| Follow-Up | [종료] | Closed |

## 6. 공급관리 대시보드

### 목적
전체 공급 주문 현황을 한눈에 파악합니다.

### 화면 구성
- **요약 카드**: 전체 / 배송 중 / 예정일 초과 / 출고 확정
- **주문 목록**: 배송 예정일 초과 항목은 주황색 배경으로 강조
- **필터**: 환자명/케이스 ID 검색, 상태 필터

> **중요**: 이 화면은 읽기 전용입니다. 주문 등록·수정은 케이스 상세에서 합니다.

**항목 클릭** → 해당 케이스 상세의 주문/배송 섹션으로 바로 이동

## 7. 시술일정 대시보드

### 목적
시술 예정 케이스와 일정 확정 대기 케이스를 관리합니다.

### 화면 구성

**섹션 1: 시술 예정** (Treatment Scheduled 상태)
- D-day 카운트다운 표시
- 가장 임박한 일정 순 정렬

**섹션 2: 일정 확정 대기** (Acceptance Confirmed 상태)
- 시술 일정이 아직 지정되지 않은 케이스
- **[시술 일정 확정]** 버튼 클릭 → 케이스 상세에서 날짜 입력

> **중요**: 이 화면은 읽기 전용입니다. 시술일 입력은 케이스 기본정보 수정에서 합니다.

## 8. 결제/청구 대시보드

### 목적
전체 청구·납부 현황을 조회합니다.

### 화면 구성
- **요약 카드**: 전체 / 납부 완료 / 부분납부 / 미납
- **청구 목록**: 상태별 배지로 표시 (완납/부분납부/환불/미납)
- **필터**: 환자명/케이스 ID 검색, 납부 상태 필터

> **중요**: 이 화면은 읽기 전용입니다. 청구 생성·납부 처리는 케이스 상세에서 합니다.

**항목 클릭** → 해당 케이스 상세의 결제/청구 섹션으로 바로 이동

## 9. 추적관리 대시보드

### 목적
기한 초과 추적관찰을 즉시 파악하고 예정 일정을 관리합니다.

### 화면 구성
- **긴급 알림**: 기한 초과 건수가 있으면 상단에 빨간 배너 표시
- **요약 카드**: 전체 / 기한 초과 / 7일 내 예정 / 완료
- **즉시 처리 필요 섹션**: 기한 초과 항목만 별도로 강조
- **전체 목록**: 단계(D7/D14/D30/D90/D180) 및 완료 여부 필터

> **중요**: 이 화면은 읽기 전용입니다. 완료 처리는 케이스 상세에서 합니다.

**항목 클릭** → 해당 케이스 상세의 추적관리 섹션으로 바로 이동

## 11. 워크플로 전체 흐름

```
1. 리드 접수
   └─ 리드관리에서 등록

2. 케이스 전환
   └─ 리드 → 케이스 전환 (병원·코디 배정)

3. 상담 진행
   └─ 케이스 상태: Consulting

4. 승인 확정
   └─ 케이스 상태: Acceptance Confirmed
   └─ 시술일정 대시보드에 "일정 확정 대기"로 표시

5. 병원 검토
   └─ 케이스 상태: Under Hospital Review
   └─ 케이스 상세 > 병원 검토 섹션에서 결과 기록

6. 시술 일정 확정
   └─ 케이스 기본정보에서 시술 예정일 입력
   └─ 시술일정 대시보드에 D-day와 함께 표시

7. 공급 주문 (필요시)
   └─ 케이스 상세 > 주문/배송 섹션에서 등록
   └─ 공급관리 대시보드에서 현황 확인

8. 청구 생성 (필요시)
   └─ 케이스 상세 > 결제/청구 섹션에서 청구 생성
   └─ 결제/청구 대시보드에서 현황 확인

9. 시술 완료
   └─ 케이스 상태: Treatment Done

10. 추적관리
    └─ 케이스 상태: Follow-Up
    └─ D7 → D14 → D30 → D90 → D180 순서로 완료 처리
    └─ 추적관리 대시보드에서 기한 초과 모니터링

11. 케이스 종료
    └─ 케이스 상태: Closed
```

## 자주 묻는 질문

**Q. 케이스 ID는 어떤 형식인가요?**  
A. `MSO-YYYY-NNN` 형식입니다 (예: `MSO-2026-042`). 화면에는 `환자이름-NNN` 형식으로 표시됩니다 (예: `야마다타로-042`).

**Q. 공급 주문을 등록하면 왜 바로 "배송 중"이 되나요?**  
A. 오프라인에서 이미 발주가 확인된 상태로 등록하는 경우가 대부분이어서, 등록 즉시 배송 중으로 처리됩니다. 실제 배송 도착 확인 시 "출고 확정" 버튼을 누르면 됩니다.

**Q. 견적서나 청구서는 시스템에서 발행할 수 없나요?**  
A. 오프라인으로 처리합니다. 시스템에서는 납부 완료 여부(일시납부/부분납부/환불)만 기록합니다.

**Q. 추적관리 기한이 초과되면 어떻게 되나요?**  
A. 추적관리 대시보드 상단에 빨간 알림이 표시되고, "즉시 처리 필요" 섹션에 강조 표시됩니다. 항목을 클릭하면 해당 케이스로 이동하여 완료 처리할 수 있습니다.

**Q. 케이스 상태를 이전 단계로 되돌릴 수 있나요?**  
A. 상태 전환은 앞으로만 진행됩니다. 잘못 전환한 경우 MSO Admin에게 문의하세요.
